# WRF U-Net Downscaling Teaching Bundle

This teaching bundle is a self-contained notebook workflow for weather-field
downscaling with a small U-Net. It starts from compact WRF NPZ files, prepares
model-ready tensors, trains a residual downscaling model, evaluates it, and
saves prediction files for classroom inspection.

The main notebook is:

- `WRF_UNet_Training_Testing_Colab.ipynb`

Open and run the notebook from this directory so the local imports
(`config.py`, `Network.py`, and `plot_utils.py`) and relative data paths resolve
correctly.

## What The Bundle Teaches

- How coarse and fine WRF fields are paired for supervised downscaling.
- How coarse fields and static fields are resized onto a common `48 x 48`
  teaching grid.
- How rainfall is transformed as `log(precipitation + 0.001)`.
- How a U-Net predicts normalized residuals instead of predicting the fine
  field directly.
- How to reconstruct fine-resolution predictions from predicted residuals.
- How to save processed datasets, checkpoints, plots, and prediction NPZ files.

## Quick Start

1. Clone or copy this folder.
2. Manually download or copy the required raw NPZ files into `data/`.
3. Open `WRF_UNet_Training_Testing_Colab.ipynb` in Colab, JupyterLab, or VS Code.
4. Make sure the notebook working directory is this folder:

   ```python
   %cd /path/to/dl_downscaling_teaching
   ```

5. Run the notebook from top to bottom.

The notebook creates `outputs/` automatically and writes processed data files
under `data/`.

Raw NPZ files are treated as external teaching data. In a real teaching setup,
students should receive or download them from the course data source and place
them in `data/`; they are not expected to be downloaded from GitHub.

## Important Files

- `WRF_UNet_Training_Testing_Colab.ipynb`  
  The main teaching notebook. It contains preprocessing, dataset construction,
  training, testing, checkpoint loading, inference, and visualization.

- `config.py`  
  Central configuration for paths, weather variables, static variables,
  normalization statistics, U-Net settings, training settings, and plotting
  bounds.

- `Network.py`  
  U-Net implementation used by the notebook. The teaching configuration builds a
  compact U-Net with 8 input channels and 6 output channels.

- `plot_utils.py`  
  Plotting helpers for loss curves and weather maps. Weather maps use the
  configured Hong Kong-region latitude-longitude domain.

- `data/wrf_static_raw.npz`  
  Static-field input expected by the notebook. This file contains real static
  arrays, not placeholders. It provides `LSM` and `HGT` on the fine WRF grid,
  along with latitude and longitude arrays.

## Data Files

Before running the notebook, manually place these raw NPZ files in `data/`:

- `data/wrf_train_coarse_raw.npz`
- `data/wrf_train_fine_raw.npz`
- `data/wrf_test_coarse_raw.npz`

Raw train/test files use a shared structure:

- `data`: weather tensor with shape `(time, variable, y, x)`
- `time`: timestamps
- `lat`, `lon`: grid coordinates
- `variable_names`: weather-channel names
- metadata such as `years`, `source_files`, `domain`, and `resolution`

Expected raw file shapes:

| File | Shape |
| --- | --- |

| `data/wrf_train_fine_raw.npz` | `(1648, 6, 43, 39)` |
| `data/wrf_test_coarse_raw.npz` | `(824, 6, 15, 12)` |

The six weather variables are configured in `config.py`:

- `T2`
- `TD2`
- `MSLP`
- `U10`
- `V10`

The two static variables are:

- `LSM`
- `HGT`

## Notebook Workflow

The main notebook is organized as a complete teaching sequence:

1. Set up paths and output directories.
2. Import local modules and scientific Python packages.
3. Define helper functions for loading, resizing, normalizing, and reconstructing
   fields.
4. Plot sample coarse and fine weather variables.
5. Load raw train/test NPZ files and normalization statistics.
6. Resize coarse, fine, and static fields to the `48 x 48` teaching grid.
7. Transform precipitation using `log(precipitation + 0.001)`.
8. Build residual targets as `fine - coarse`.
9. Normalize weather inputs and residual targets.
10. Save processed train/test NPZ files.
11. Load processed files through a lightweight dataset class.
12. Train the U-Net.
13. Save the best checkpoint and loss curve.
14. Reload the best checkpoint for inference.
15. Save coarse, predicted, and fine test fields for comparison.

## Generated Processed Files

Running the preprocessing cells creates:

- `data/wrf_train_processed.npz`
- `data/wrf_test_processed.npz`

Each processed file stores:

- `inputs`: normalized model input tensor, including static channels
- `targets`: normalized fine-minus-coarse residual tensor
- `coarse`: resized coarse weather fields in physical units
- `fine`: resized fine weather fields in physical units
- `time`: timestamps
- `lat`, `lon`: `48 x 48` teaching-grid coordinates

Expected processed shapes:

| File | `inputs` | `targets`, `coarse`, `fine` |
| --- | --- | --- |
| `wrf_train_processed.npz` | `(1648, 8, 48, 48)` | `(1648, 6, 48, 48)` |
| `wrf_test_processed.npz` | `(824, 8, 48, 48)` | `(824, 6, 48, 48)` |

The 8 input channels are 6 normalized weather channels plus `LSM` and `HGT`.
The 6 output channels are normalized residuals for the weather variables.

## Generated Outputs

Running training and inference creates files under `outputs/`:

- `unet_npz_best.pt`: best model checkpoint by test loss
- `unet_loss_curve.png`: train/test loss curve
- `wrf_test_coarse.npz`: resized coarse test fields
- `wrf_test_predicted.npz`: U-Net downscaled predictions
- `wrf_test_fine.npz`: resized fine test fields used as reference

The inference output NPZ files are intended for side-by-side comparison of
coarse input, predicted fine field, and fine reference field.

## Configuration Notes

Most classroom knobs are in `config.py`:

- `NUM_EPOCHS`
- `BATCH_SIZE`
- `LEARNING_RATE`
- `HIGH_RES_SHP`
- `IN_OUT_PARAMS`
- `CONSTANT_PARAMS`
- `MODEL_KWARGS`
- `RAW_STATS`
- `RESIDUAL_STATS`

For a faster demo, reduce `NUM_EPOCHS`. For a more thorough training run,
increase `NUM_EPOCHS` or adjust the model size in `MODEL_KWARGS`.

## Static Fields

`data/wrf_static_raw.npz` is manually supplied with the raw weather files and
should be placed in `data/` before running the notebook. It contains:

- `lat`
- `lon`
- `LSM`
- `HGT`

During preprocessing, `LSM` is resized with nearest-neighbor interpolation, while
`HGT` is resized with bilinear interpolation and normalized before being
appended to every timestep.

## Troubleshooting

- `ModuleNotFoundError: No module named 'config'`  
  Run the notebook from the `dl_downscaling_teaching` directory.

- `FileNotFoundError` for files in `data/`  
  Confirm the manually downloaded raw NPZ files and `wrf_static_raw.npz` are
  present with the exact names listed above.

- `ModuleNotFoundError: No module named 'cartopy'`  
  Install `cartopy` in the active notebook environment, then rerun the plotting
  cells.

- Training is slow on CPU  
  Use a GPU runtime in Colab or reduce `NUM_EPOCHS` in `config.py`.
