# WRF U-Net Downscaling Teaching Bundle

This repository contains a notebook-based teaching workflow for downscaling
Weather Research and Forecasting (WRF) model output over the Pearl River Delta
(PRD) region. The workflow uses paired coarse-resolution and fine-resolution WRF
fields to train a compact residual U-Net.

The model receives coarse WRF weather variables plus static geographic fields
and learns the correction needed to reconstruct the fine-resolution reference:

```text
predicted fine field = matched coarse field + predicted residual
```

Use the notebook that matches your environment:

- `WRF_UNet_Training_Testing.ipynb`: local computer version
- `WRF_UNet_Training_Testing_Colab.ipynb`: Google Colab version

## What You Will Learn

- How coarse and fine WRF fields are paired for supervised downscaling.
- How atmospheric fields can be treated as image-like tensors.
- How coarse and fine grids are resized onto a common `48 x 48` teaching grid.
- Why precipitation is transformed with `log(precipitation + 0.001)`.
- How static fields such as land-sea mask and terrain height are added as model
  inputs.
- How a U-Net predicts normalized residuals rather than full fine fields.
- How to reconstruct and save coarse, predicted, and fine comparison files.

Expected raw file shapes:

| File | Expected shape |
| --- | --- |
| `wrf_train_coarse_raw.npz` | `(1648, 6, 15, 12)` |
| `wrf_train_fine_raw.npz` | `(1648, 6, 43, 39)` |
| `wrf_test_coarse_raw.npz` | `(824, 6, 15, 12)` |
| `wrf_test_fine_raw.npz` | `(824, 6, 43, 39)` |
| `wrf_static_raw.npz` | `LSM` and `HGT`: `(43, 39)` |

The weather NPZ files should contain:

- `data`: array shaped `(time, variable, y, x)`
- `time`: timestamps
- `lat`, `lon`: coordinates
- `variable_names`: names for the weather channels

The static NPZ file should contain:

- `lat`
- `lon`
- `LSM`
- `HGT`

## Weather Variables

The six weather channels are defined in `config.py`:

- `T2`: 2-m air temperature
- `TD2`: 2-m dew point temperature
- `MSLP`: mean sea level pressure
- `U10`: 10-m east-west wind component
- `V10`: 10-m north-south wind component
- `ACC_6H_PRECIP`: 6-hour accumulated precipitation

`ACC_6H_PRECIP` is loaded from the raw files in millimeters. During
preprocessing, the notebook transforms it with:

```text
log(precipitation + 0.001)
```

The model trains in this transformed space. The final saved prediction NPZ files
convert precipitation back to millimeters.

## Model Inputs And Outputs

After preprocessing, the model uses:

| Tensor | Shape | Meaning |
| --- | --- | --- |
| `inputs` | `(N, 8, 48, 48)` | 6 normalized weather channels + `LSM` + `HGT` |
| `targets` | `(N, 6, 48, 48)` | normalized fine-minus-coarse residuals |
| `coarse` | `(N, 6, 48, 48)` | matched coarse fields |
| `fine` | `(N, 6, 48, 48)` | matched fine reference fields |

`N` is the number of timesteps. The training set has 1648 samples and the test
set has 824 samples when using the expected teaching data.

The compact U-Net is configured in `config.py` with:

- 8 input channels
- 6 output channels
- `model_channels = 32`
- `channel_mult = [1, 2, 2]`
- `num_blocks = 1`
- `NUM_EPOCHS = 20`
- `BATCH_SIZE = 8`
- `LEARNING_RATE = 0.00003`

## Generated Outputs

Training and inference create these files under `outputs/`:

- `unet_npz_best.pt`: best model checkpoint based on test loss
- `unet_loss_curve.png`: training and test loss curve
- `wrf_test_coarse.npz`: matched coarse 2023 test fields
- `wrf_test_predicted.npz`: U-Net downscaled predictions
- `wrf_test_fine.npz`: matched fine 2023 reference fields

The three comparison NPZ files use this structure:

- `data`: weather tensor with shape `(time, variable, 48, 48)`
- `time`: timestamps
- `lat`: 48 latitude values
- `lon`: 48 longitude values
- `variable_names`: `T2`, `TD2`, `MSLP`, `U10`, `V10`, `ACC_6H_PRECIP`

In these saved output files, `ACC_6H_PRECIP` is converted back to millimeters.

## Troubleshooting

`ModuleNotFoundError: No module named 'config'`

Run the notebook from inside the `dl_downscaling_teaching` folder. In Colab,
check that the setup cell successfully ran:

```python
%cd /content/drive/MyDrive/dl_downscaling_teaching
```

`FileNotFoundError` for a file in `data/`

Check that all five raw NPZ files are present with the exact names listed in
this README. In Colab, run `!ls data` after mounting Drive.

`ModuleNotFoundError: No module named 'cartopy'`

Install Cartopy in the active environment. The Colab notebook already includes
`!pip install -q cartopy` in section 0.

`Using device: cpu`

The notebook did not detect a GPU. In Colab, go to `Runtime -> Change runtime
type -> Hardware accelerator -> GPU`, then rerun the notebook from the start.

Training is too slow

Use a Colab GPU runtime or reduce `NUM_EPOCHS` in `config.py`.
